<h1 align="center">pixel: Node-based <i>everything</i>.</h1>
Pixel aims to make everything from document editing to graphic designing
easier, streamlined, and reactive through flowing, elegant node networks.

## Roadmap

### Version 0.1.0 (Concepts)
- Cairo/Pango user interface with GTK+ window creation.
- Node-based RTF editing.

### Version 0.2.0 (UI Improvements)
- Zoomable and scrollable workspace.
- Adjustable user interface settings.
- More editing functionality (ex. align nodes, group nodes, panes)

### Version 0.3.0 (Plugins)
- Introduce multi-language plugin base.
- Create core plugins in C, and language plugins for popular languages.
- Allow plugins to draw custom nodes through GTK+, Qt, HTML, Tk, or basic XML-based layout.

### Version 0.4.0 (GTK+ Integration)
- Create custom GTK+ widgets to split and drag panes. Add in a UI node to the editor.
- Save custom settings files.

### Version 0.5.0 (Essential Functionality)
- Add save actions (.px file format).
- Add save templates.
- Add undo tree.
- Introduce meta-nodes.

### Version 0.6.0 (Cloud)
- Launch `pixel.brandongong.org` with accounts, package manager, auto update checks, etc.

### Version 0.7.0 (PixelOffice)
- Build PixelOffice core packages to match LibreOffice and MS Word.

### Version 0.8.0 (Fine-tune)
- Bug fixes.
- Easter eggs.
- Platform-specific performance testing.

### Version 1.0.0
- Initial release.
